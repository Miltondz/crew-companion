# 3.1 — Surface Registry: Runtime Foundations (Design Proposal)

**Author:** crew-architect
**Date:** 2026-05-10
**Status:** Proposal (awaiting implementer in block 3.1)
**Scope:** Phase A, block 3.1 of `project-docs/MASTER_WORK_PLAN.md`
**Out of scope (deferred to later blocks):**
- 3.2 Spatial Grammar — Layout Engine, region negotiation, pinning persistence
- 3.3 Capability Engine — `Capability` enum, policy gates, audit log
- 3.4 Persistence — DB-backed pinning, workspace state
- 3.5 Envelope protocol — BFF transport + cross-language sync of envelope schema

This block builds **only** the frontend Surface Registry runtime: manifests, registry class,
SurfaceHost, the new envelope shape on the frontend, and a backward-compat adapter so the
existing `renderSurface` tool keeps working.

---

## 1. Decision

Replace `SurfaceRenderer.tsx` (switch/case stub) with a manifest-driven **Surface Registry**
living under `apps/frontend/src/runtime/surface-registry/`. Surfaces self-describe via
co-located `manifest.ts` files; the registry resolves intents against context, validates
envelopes with Zod, and lazy-loads the component via React 19 `use()`.

The new `SurfaceEnvelope` is the **full** envelope from MASTER_WORK_PLAN Part 3.5 (with
`envelopeId`, `agentId`, `emittedAt`, `intent`, `priority`, `context`, etc.), but block
3.1 also ships a **backward-compat adapter** (`adaptLegacyEnvelope`) that lifts the
current `{type, payload}` shape into a full envelope using runtime-derived context. This
lets block 3.1 ship without waiting on the BFF refactor in 3.5.

Capabilities are typed as `string[]` in this block; the `Capability` enum and policy
checks land in 3.3. The registry validates manifest-declared capabilities are present
but treats them as opaque strings until 3.3 wires the gate.

`preferredZone` is typed as a `RegionId` placeholder string union covering the six zones
named in MASTER_WORK_PLAN 3.2.2. 3.2 will own that union and the routing semantics; 3.1
just records the surface's preference.

---

## 2. Rationale

**Why a Map-backed registry over module-graph auto-discovery?**
Explicit `register()` calls in `bootstrap.ts` give a single chokepoint for validation
errors and a stable load order. Auto-discovery (e.g. Vite glob imports) would couple the
runtime to bundler magic and silently swallow missing manifests. The Master Work Plan's
*operational clarity* test wins over the *DX terseness* test here.

**Why React 19 `use()` over `React.lazy` + manual Suspense boundaries?**
`use()` lets `SurfaceHost` read a cached promise directly and integrate with the parent
`<Suspense>` already mounted by the page. Each surface gets a stable promise per-manifest
(memoized once) so concurrent mounts dedupe. `React.lazy` would force every surface into
its own boundary, which fights the Layout Engine's intent in 3.2 of mounting multiple
surfaces in one region with shared skeleton transitions.

**Why ship the full envelope shape now if BFF can't emit it yet?**
Designing the registry around the legacy `{type, payload}` shape would force a second
refactor in 3.5. Defining the full type now + an adapter for the old shape costs ~20 LOC
and unblocks 3.5 to slot in without touching the registry.

**Why keep `id` and `intent` separate?**
A surface's `id` is its stable identity (`'task_suggestion'`). An `intent` is what the
agent semantically asks for (`'render_surface'`, `'show_ambient_alert'`, ...). One intent
can resolve to many surfaces depending on `context`. In 3.1 we only ship the
`render_surface` intent and a 1:1 `id`-to-surface mapping, but the indirection is the seam
that lets 3.2/3.3 add routing without changing the call sites.

**Why Zod for envelopeSchema?**
Already a project dep (`zod@^3.25.76`). Output type infers cleanly into the surface's
`Props['payload']` so we get static + runtime guarantees from one declaration.

### 4.1 `apps/frontend/src/runtime/surface-registry/types.ts`

```typescript
import type { ComponentType } from 'react'
import type { z, ZodTypeAny } from 'zod'
import type { Role, TechnicalLevel, UrgencyPhase } from '@/lib/crew/types'

// ───────────────────────────────────────────────────────────────────────────
// Region IDs — placeholder union owned by 3.2. Listed here so manifests
// can declare `preferredZone` without depending on a not-yet-existent file.
// 3.2 will export the canonical `RegionId` from
// `apps/frontend/src/runtime/workspace/regions.ts` and 3.1's copy will be
// re-exported from there (single source of truth).
// ───────────────────────────────────────────────────────────────────────────
export type RegionId =
  | 'command-surface'
  | 'primary-workzone'
  | 'context-rail'
  | 'agent-rail'
  | 'activity-stream'
  | 'ambient-overlay'

// ───────────────────────────────────────────────────────────────────────────
// Capability — typed as opaque string in 3.1. 3.3 will replace with enum.
// Keep import path stable so 3.3's swap is a one-line change here.
// ───────────────────────────────────────────────────────────────────────────
export type Capability = string

// ───────────────────────────────────────────────────────────────────────────
// SurfaceIntent — extensible literal union. 3.1 ships `render_surface` only.
// ───────────────────────────────────────────────────────────────────────────
export type SurfaceIntent =
  | 'render_surface'
  // Reserved for later blocks — not handled by registry in 3.1:
  // | 'set_mascot_mood'
  // | 'highlight_tasks'
  // | 'update_state'
  // | 'request_approval'
  // | 'show_ambient_alert'

export type EnvelopePriority = 'low' | 'medium' | 'high' | 'critical'

// ───────────────────────────────────────────────────────────────────────────
// RuntimeContext — what the registry needs to resolve an intent.
// Mirrors the agent-side context in MASTER_WORK_PLAN 3.5.1.
// ───────────────────────────────────────────────────────────────────────────
export interface RuntimeContext {
  role: Role
  techLevel: TechnicalLevel
  phase: UrgencyPhase
  hasActiveBlocker: boolean
  workspaceId: string
}

// ───────────────────────────────────────────────────────────────────────────
// SurfaceEnvelope — full envelope shape (Phase A target, per MWP 3.5.1).
// In 3.1 the BFF still emits legacy `{type, payload}`; `adaptLegacyEnvelope`
// lifts it to this shape using RuntimeContext from the current page.
// ───────────────────────────────────────────────────────────────────────────
export interface SurfaceEnvelope<TPayload = unknown> {
  // Identity
  envelopeId: string                        // uuid
  agentId: string                           // 'orchestrator' | 'legacy' | …
  emittedAt: number                         // epoch ms

  // Intent
  intent: SurfaceIntent
  priority: EnvelopePriority

  // The actual surface body — validated by manifest.envelopeSchema
  // The discriminator that picks the manifest. For `render_surface`,
  // this is the surface `id` (e.g. 'task_suggestion').
  surfaceId: string
  payload: TPayload

  // Snapshot of context the agent saw when emitting
  context: RuntimeContext

  // Capability claims the agent declares it needs to mount this surface
  requiredCapabilities: Capability[]

  // Lifecycle hints (registry stores; 3.2 Layout Engine honors)
  hibernatable: boolean
  pinnable: boolean
  ephemeral?: number                        // ms before auto-dismiss
}

// ───────────────────────────────────────────────────────────────────────────
// Legacy envelope — the current `{type, payload}` shape emitted by the BFF.
// Kept here so the adapter has a typed source and we can grep for callers.
// ───────────────────────────────────────────────────────────────────────────
export interface LegacySurfaceEnvelope {
  type: string                              // == surfaceId in new shape
  payload: Record<string, unknown>
}

// ───────────────────────────────────────────────────────────────────────────
// SurfaceProps — every surface component MUST accept this exact shape.
// `payload` is the Zod-validated body. `context` lets the surface react
// to role/phase for in-component variants (e.g. low-tech wording).
// ───────────────────────────────────────────────────────────────────────────
export interface SurfaceProps<TPayload = unknown> {
  payload: TPayload
  context: RuntimeContext
  envelope: SurfaceEnvelope<TPayload>       // full envelope for advanced uses
}

// ───────────────────────────────────────────────────────────────────────────
// SurfaceManifest — single source of truth for a surface.
// ───────────────────────────────────────────────────────────────────────────
export interface SurfaceManifest<TSchema extends ZodTypeAny = ZodTypeAny> {
  // ── Identity ─────────────────────────────────────────────────────────
  id: string                                // e.g. 'task_suggestion'
  version: string                           // semver
  displayName: string                       // human-readable, for debug

  // ── Authorization (validated in 3.3) ─────────────────────────────────
  requiredCapabilities: Capability[]        // opaque strings in 3.1

  // ── Visibility filters (consumed by resolver) ────────────────────────
  visibleToRoles: Role[]                    // empty array = all
  visibleToTechLevels?: TechnicalLevel[]    // undefined = all
  forbiddenInPhases?: UrgencyPhase[]        // e.g. ['panic'] for non-critical

  // ── Input contract ───────────────────────────────────────────────────
  envelopeSchema: TSchema                   // Zod schema for `payload`

  // ── Visual properties (consumed by 3.2 Layout Engine) ────────────────
  density: 'compact' | 'standard' | 'hero'
  preferredZone: RegionId
  minWidth?: number                         // px
  minHeight?: number

  // ── Lifecycle hooks ──────────────────────────────────────────────────
  canPin: boolean
  hibernatable: boolean
  priority: number                          // 0-100; higher wins conflicts

  // ── Lazy loader ──────────────────────────────────────────────────────
  load: () => Promise<{
    default: ComponentType<SurfaceProps<z.infer<TSchema>>>
  }>
}

// ───────────────────────────────────────────────────────────────────────────
// Validation result — narrow wrapper around Zod's SafeParseReturnType so
// callers don't need to know about Zod internals.
// ───────────────────────────────────────────────────────────────────────────
export type ValidationResult<T = unknown> =
  | { success: true; data: T }
  | { success: false; error: { issues: Array<{ path: (string | number)[]; message: string }> } }

// ───────────────────────────────────────────────────────────────────────────
// Mount result — what `mount()` returns to SurfaceHost.
// ───────────────────────────────────────────────────────────────────────────
export interface MountedSurface<TPayload = unknown> {
  manifest: SurfaceManifest
  Component: ComponentType<SurfaceProps<TPayload>>
}

// ───────────────────────────────────────────────────────────────────────────
// Registry error reasons — discriminated union so SurfaceHost can render
// helpful error cards.
// ───────────────────────────────────────────────────────────────────────────
export type ResolveFailure =
  | { reason: 'unknown_surface'; surfaceId: string }
  | { reason: 'role_mismatch'; surfaceId: string; required: Role[]; got: Role }
  | { reason: 'tech_level_mismatch'; surfaceId: string }
  | { reason: 'forbidden_in_phase'; surfaceId: string; phase: UrgencyPhase }
  | { reason: 'missing_capabilities'; surfaceId: string; missing: Capability[] }
```

### 4.2 `apps/frontend/src/runtime/surface-registry/registry.ts`

```typescript
import type {
  Capability,
  MountedSurface,
  ResolveFailure,
  RuntimeContext,
  SurfaceEnvelope,
  SurfaceIntent,
  SurfaceManifest,
  ValidationResult,
} from './types'

export class SurfaceRegistry {
  /**
   * Register a manifest. Throws on duplicate id or invalid manifest shape.
   * Idempotent only for byte-identical manifest objects (rejects re-registration
   * with the same id but different content — prevents hot-reload drift).
   */
  register(manifest: SurfaceManifest): void

  /**
   * Look up a manifest by surface id. Returns null if not registered.
   * Does NOT apply visibility / capability filters — used by debug tools
   * and the legacy adapter.
   */
  get(surfaceId: string): SurfaceManifest | null

  /**
   * Resolve an intent + context to a single manifest, or a failure reason.
   *
   * In 3.1 only `'render_surface'` is handled; the envelope MUST carry
   * `surfaceId` and the resolver returns either that manifest (if visible)
   * or a typed failure. Later blocks may add intent → many-surfaces routing.
   */
  resolve(
    intent: SurfaceIntent,
    surfaceId: string,
    context: RuntimeContext,
    requiredCapabilities: Capability[],
    grantedCapabilities: Capability[],   // in 3.1: pass `requiredCapabilities` (no-op)
  ): { ok: true; manifest: SurfaceManifest } | { ok: false; failure: ResolveFailure }

  /**
   * Validate a raw payload against a manifest's Zod schema.
   * Pure delegation to Zod's `safeParse` with a normalized error shape.
   */
  validate<T>(payload: unknown, manifest: SurfaceManifest): ValidationResult<T>

  /**
   * Trigger the lazy loader for a manifest. Returns a stable cached promise
   * (one per manifest, memoized for the lifetime of the registry) so React 19
   * `use()` reads consistent identity across renders and Suspense doesn't
   * thrash.
   */
  loadPromise<T>(manifest: SurfaceManifest): Promise<MountedSurface<T>>

  /**
   * Convenience: resolve + validate + load in one call. Used by SurfaceHost.
   * Returns a discriminated union covering every failure mode.
   */
  mount<T>(
    envelope: SurfaceEnvelope<T>,
    context: RuntimeContext,
    grantedCapabilities: Capability[],
  ):
    | { status: 'ready'; manifest: SurfaceManifest; promise: Promise<MountedSurface<T>>; data: T }
    | { status: 'resolve_failed'; failure: ResolveFailure }
    | { status: 'validation_failed'; error: ValidationResult<T>['error' & {}] }

  /** Test helper. Clears all registered manifests. Do NOT call in production. */
  __resetForTests(): void

  /** Debug helper. Returns all registered manifest ids. */
  listIds(): string[]
}

/** Singleton — imported wherever the registry is needed. */
export const surfaceRegistry: SurfaceRegistry
```

### 4.3 `apps/frontend/src/runtime/surface-registry/adapter.ts`

```typescript
import type {
  LegacySurfaceEnvelope,
  RuntimeContext,
  SurfaceEnvelope,
} from './types'

/**
 * Lift the BFF's current `{type, payload}` envelope into a full SurfaceEnvelope.
 *
 * Generates a fresh envelopeId, stamps `agentId: 'legacy'`, sets `intent` to
 * `'render_surface'`, and copies `type` into both `surfaceId` (for resolution)
 * and a synthetic context taken from the page's RuntimeContext.
 *
 * Lifecycle hints default conservatively:
 *   - `priority: 'medium'`
 *   - `hibernatable: true`
 *   - `pinnable: true`
 *   - `requiredCapabilities: []`  (3.3 will populate from the manifest)
 *
 * Removed when block 3.5 lands and the BFF emits full envelopes natively.
 */
export function adaptLegacyEnvelope(
  legacy: LegacySurfaceEnvelope,
  context: RuntimeContext,
): SurfaceEnvelope

/**
 * Inverse — lets the legacy `renderSurface` tool's Zod parameter schema stay
 * exactly as today (`{envelope: {type, payload}}`) while we shovel it through
 * the new registry. Used by the temporary tool render slot.
 */
export function isLegacyEnvelope(
  e: unknown,
): e is LegacySurfaceEnvelope
```

### 4.4 `apps/frontend/src/runtime/surface-registry/SurfaceHost.tsx`

```typescript
'use client'

import { use, Suspense } from 'react'
import { surfaceRegistry } from './registry'
import type { RuntimeContext, SurfaceEnvelope } from './types'

interface SurfaceHostProps {
  envelope: SurfaceEnvelope
  context: RuntimeContext
  /** Defaults to []. 3.3 fills in from the session. */
  grantedCapabilities?: string[]
}

/**
 * Decision tree (executed top-down, first failure renders):
 *
 *  1. registry.mount(envelope, context, grantedCapabilities)
 *  2. status === 'resolve_failed'    →  <UnsupportedSurfaceCard failure={...} />
 *  3. status === 'validation_failed' →  <InvalidEnvelopeCard error={...} />
 *  4. status === 'ready':
 *       <Suspense fallback={<SurfaceSkeleton density={manifest.density} />}>
 *         <SurfaceLazyMount promise={...} data={...} context={...} envelope={...} />
 *       </Suspense>
 *     where SurfaceLazyMount does:
 *       const { Component } = use(promise)
 *       return <Component payload={data} context={context} envelope={envelope} />
 *
 *  Errors thrown inside the surface itself are caught by the page-level
 *  ErrorBoundary (added in 4.7); this component does not own that boundary
 *  because 3.2's Layout Engine wants per-region boundaries.
 */
export function SurfaceHost(props: SurfaceHostProps): JSX.Element

// Internal — exported only for tests:
export function UnsupportedSurfaceCard(props: { failure: import('./types').ResolveFailure }): JSX.Element
export function InvalidEnvelopeCard(props: { error: NonNullable<import('./types').ValidationResult['error' & {}]> }): JSX.Element
export function SurfaceSkeleton(props: { density: 'compact' | 'standard' | 'hero' }): JSX.Element
```

## 6. Backward compatibility plan

**Problem:** Until 3.5 lands, the BFF still emits `{type, payload}` envelopes via the
`renderSurface` CopilotKit frontend tool, wired identically in three pages
(`leader/page.tsx:136`, `member/[memberId]/page.tsx:141`, `docs/page.tsx:94`).

**Strategy:** Don't change the tool's Zod parameter schema yet. Replace its `render`
callback only.

### 6.1 Pre-3.1 (current)

```typescript
useFrontendTool({
  name: 'renderSurface',
  parameters: z.object({
    envelope: z.object({ type: z.string(), payload: z.record(z.unknown()) }),
  }),
  render: ({ args }) => <SurfaceRenderer envelope={args.envelope} />,
})
```

### 6.2 Post-3.1 (this block ships)

```typescript
useFrontendTool({
  name: 'renderSurface',
  parameters: z.object({
    envelope: z.object({ type: z.string(), payload: z.record(z.unknown()) }),
    //  ^^ schema unchanged so the agent doesn't need a redeploy
  }),
  render: ({ args }) => {
    const context = useRuntimeContext()   // small hook over the page's state
    const fullEnvelope = adaptLegacyEnvelope(args.envelope, context)
    return <SurfaceHost envelope={fullEnvelope} context={context} />
  },
})
```

The three pages each get the same 4-line change. The `useRuntimeContext` hook lives in
`apps/frontend/src/runtime/surface-registry/useRuntimeContext.ts` and reads:
- `role` from page-level constant (leader page → `'leader'`, member page → `'member'`, docs → `'leader'` as a default for 3.1; 3.2 will derive from auth)
- `techLevel` from the active member, defaulting to `'high-tech'`
- `phase` from `state.urgencyPhase`
- `hasActiveBlocker` from `state.blockers.some(b => !b.resolved)`
- `workspaceId` placeholder `'default'` until 4.1 lands auth

### 6.3 Post-3.5 (future, NOT this block)

The agent emits full `SurfaceEnvelope` objects. The tool's Zod schema swaps to the full
shape. `adaptLegacyEnvelope` and `isLegacyEnvelope` are deleted. The hook + `SurfaceHost`
usage stays identical.

## 7. Invariants the implementer MUST preserve

Numbered checklist — every PR in this block is reviewed against these.

1. **Manifest is the only source of truth.** A surface MUST NOT be reachable by any code path that doesn't go through its manifest. No `import { TaskSuggestionPanel } from '...'` followed by direct JSX from a page.
2. **Envelopes are validated before component code runs.** `SurfaceHost` MUST call `registry.validate` (via `mount`) before triggering `load`. Loading a surface to render an error is a defect.
3. **Lazy means lazy.** The `load` thunk MUST be a dynamic `import()`. No top-level imports of surface components inside `bootstrap.ts` or the registry. Verified by checking the Next.js build output: each surface produces its own chunk.
4. **Manifests are registered exactly once per id.** Re-registration with a different object throws (prevents hot-reload bugs masking duplicate-id errors).
5. **The registry has no React imports.** `registry.ts` is pure TS. React-aware code lives in `SurfaceHost.tsx`. This keeps the registry trivially unit-testable without a DOM.
6. **`SurfaceProps` shape is closed.** Adding fields to `SurfaceProps` is a breaking change for every surface; do it intentionally, not opportunistically.
7. **Backward-compat adapter is the only place that synthesizes envelope fields.** No page-level code may fabricate an `envelopeId` or `emittedAt`. When 3.5 deletes the adapter, the codebase has zero other call sites to clean up.
8. **No capability *enforcement* in 3.1.** Pass `requiredCapabilities` as `grantedCapabilities` in the call from `SurfaceHost` so resolve never fails on missing caps. 3.3 owns the real gate. Document this with a `// TODO(3.3)` comment at the single call site.
9. **`preferredZone` is recorded, not honored.** 3.1 always renders the resolved surface inline (as today's `SurfaceRenderer` does, inside the chat tool render slot). 3.2's Layout Engine will route by zone. Do not add zone routing logic in 3.1 — it would have to be torn out.
10. **Test fixtures never leak into bootstrap.** `bootstrap.ts` MUST NOT import `__fixture_*`. Enforced via an ESLint `no-restricted-imports` rule, or at minimum a comment + reviewer checklist.
11. **Zero changes to `apps/agent` and `apps/bff` in this block.** If a registry decision requires the agent to change, push that decision to 3.5.
12. **Existing user-visible behavior is preserved.** Every surface that mounts via the legacy `renderSurface` tool today MUST mount identically after the refactor — same content, same styling, same position in the chat. The acceptance step "no regressions vs. current behavior" (MWP Phase A gate) is the hard check.
