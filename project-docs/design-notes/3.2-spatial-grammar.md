**Author:** main thread (Sonnet 4.6) — Opus credit out
**Status:** Design final, ready to implement

## Scope

Replace ad-hoc Tailwind layouts in the 3 pages with a **Layout Engine** that mounts surfaces into named **regions**. Add **pinning**. Apply phase-driven rules.

Out of scope (defer to Phase B):
- Replacing route-based pages with single WorkspaceShell (deferred — current pages stay as role-context wrappers)
- DB-backed pinning persistence (DDL exists in `005_pinning.sql`; runtime uses localStorage until 4.1 auth lands)
- Full surface migration into regions (only the 8 agent surfaces + 2 anchor surfaces mount via Layout Engine in 3.2; existing dashboard chrome remains as-is)

## Layout Engine API

```typescript
// apps/frontend/src/runtime/workspace/layout-engine.ts

class LayoutEngine {
  private state: LayoutState
  private listeners: Set<() => void> = new Set()

  constructor(initial?: Partial<LayoutState>)

  /** Returns current layout state (immutable snapshot). */
  getState(): LayoutState

  /** Subscribe to layout changes. */
  subscribe(fn: () => void): () => void

  /** Mount a surface. Engine picks region from manifest.preferredZone or envelope override. */
  mount(envelope: SurfaceEnvelope, opts?: { region?: RegionId; pinned?: boolean }): MountResult

  /** Unmount by mountId. Refuses if surface is pinned (unless force=true). */
  unmount(mountId: string, opts?: { force?: boolean }): boolean

  /** Pin/unpin a mount. Pinned mounts persist across phase transitions and cannot be evicted. */
  pin(mountId: string): void
  unpin(mountId: string): void

  /** Hibernate (collapse to icon) a mount. */
  hibernate(mountId: string): void
  wake(mountId: string): void

  /** Apply phase change rules. Called by useUrgencyPhase hook. */
  onPhaseChange(prev: UrgencyPhase, next: UrgencyPhase): void

  /** Reset to empty (mainly for tests). */
  reset(): void
}

export const layoutEngine = new LayoutEngine()
```

## Conflict resolution rules

When `mount()` is called and the target region is full:

1. **Skip if duplicate**: if another mount has same `manifestId` AND same payload hash → return existing mountId
2. **Evict hibernated first**: drop oldest hibernated mount with lower or equal priority
3. **Evict lowest priority**: drop the lowest-priority unpinned mount if its priority < new envelope priority
4. **Hibernate if equal**: if new envelope priority == lowest non-pinned, hibernate the lowest instead of evict
5. **Reject mount**: if all mounts in region are pinned with higher priority → return `{ ok: false, reason: 'region-full-all-pinned' }`

## Phase transition rules

```typescript
onPhaseChange(prev, next) {
  // Rule P1: entering panic → evict non-critical from primary-workzone
  if (next === 'panic' && prev !== 'panic') {
    primary.mounts
      .filter(m => !m.pinned && m.priority < 70)
      .forEach(m => this.unmount(m.mountId))
  }

  // Rule P2: leaving panic → unhibernate triage surfaces that auto-hibernated
  if (prev === 'panic' && next !== 'panic') {
    primary.mounts
      .filter(m => m.hibernated && m.envelope.context.phase === 'panic')
      .forEach(m => this.unmount(m.mountId, { force: true }))
  }

  // Rule P3: expired → freeze all generative regions (read-only display)
  if (next === 'expired') {
    setRegionCollapsed('primary-workzone', false) // visible but frozen
  }
}
```

## WorkspaceShell component (minimal — wraps existing pages)

```typescript
// apps/frontend/src/runtime/workspace/WorkspaceShell.tsx

export function WorkspaceShell({ children }: { children: ReactNode }) {
  const layout = useLayoutEngine()
  const phase = usePhase()

  // Wires phase changes into engine.
  useEffect(() => {
    layoutEngine.onPhaseChange(prevPhase, phase)
  }, [phase])

  // Hydrate pinning from localStorage on first mount.
  useEffect(() => {
    pinningStore.hydrateIntoEngine(layoutEngine)
  }, [])

  return (
    <div data-phase={phase} className="workspace-shell">
      <CommandSurfaceRegion mounts={layout['command-surface'].mounts} />
      <div className="workspace-main">

      </div>
      <ActivityStreamRegion mounts={layout['activity-stream'].mounts} />
      <AmbientOverlayRegion mounts={layout['ambient-overlay'].mounts} />

  )
```

1. Renders the region's container (CSS class for position/density)

## Framer Motion transitions

Mount/unmount uses `AnimatePresence`:
- enter: `opacity 0 → 1`, `translateY(8) → 0`, 220ms
- exit: `opacity 1 → 0`, `scale 1 → 0.97`, 180ms
- hibernate: morphism — width/height shrink to icon size, 280ms

Phase background:
- `data-phase` attribute on `.workspace-shell` drives CSS gradient (already declared in MASTER_WORK_PLAN; add gradient classes if not present)

## Invariants

1. Layout Engine state is the **only** source of truth for which surfaces are mounted where. No surface mounts itself directly.
2. Pinning persists in localStorage now; mirrors to DB in 4.1.
3. Anchor regions (command-surface, agent-rail, activity-stream) **never** lose their anchor surface — engine refuses to mount/unmount these.
4. Generative regions enforce capacity; mount returns `{ ok: false }` when no eviction possible.
5. Phase transitions trigger `onPhaseChange` rules immediately; no debounce.
6. No surface knows what region it lives in — the manifest's `preferredZone` is a hint, not a binding.
7. SurfaceHost remains unchanged. Layout Engine wraps each mount with `<SurfaceHost>`.
8. `useFrontendTool('renderSurface')` no longer renders the surface inline; it mounts and returns a notice card.
9. The 3 existing pages still work — they render their existing chrome as `children` of `WorkspaceShell`.
10. No new external dependencies (Framer Motion already in stack).

## Validation (7 questions)

1. Operational clarity? **YES** — explicit region model replaces implicit position-by-import.
2. Human throughput? **YES** — pinned surfaces survive phase changes; less re-finding.
3. User control? **YES** — explicit pin/unpin/hibernate user actions.
4. Cognitive load? **YES (reduces)** — fixed anchors prevent disorientation; generative-only zones are minority.
5. Strengthens runtime? **YES** — layout becomes a real engine, not Tailwind grid.
6. Extensibility? **YES** — new region = new spec entry; new surface auto-routes via preferredZone.
7. Philosophical coherence? **YES** — matches `mejoras/01_filosofia_generativa` "anchors + generative" hybrid.

All 7 pass. Design accepted.
